# Margarida 💖 — will you go out with me?

A tiny, playful dating invitation (with a runaway "No" button), themed around a
**sunset beach and cinema date**, plus mini-games starring **Margarida**. Built with
**Next.js**, exported as a static site, and deployed to **GitHub Pages**.

🌐 Live: **https://tomas6241.github.io/**

---

## ✨ What's inside

- **The ask** (`/`) — a sunset-beach invitation with a **Sim / Não** choice. The
  **Não** button runs away while the illustrated beach-and-cinema postcard reacts.
- **Animated beach theme** — a breathing sunset, drifting clouds and birds, sea
  glints, layered waves, sand, a beach umbrella, picnic blanket, popcorn and tickets.
  Animations use transform/opacity and include a reduced-motion fallback.
- **Joguinhos** (`/jogos`) — a menu to pick a mini-game:
  - 💗 **Apanha os Corações** — a Pac-Man-style game reskinned so Margarida collects
    hearts and turns the tables on *Quiet Place*-style creatures.
  - 🏃‍♀️ **A Fuga da Margarida** — an HTML5 canvas endless runner: run, jump, collect
    hearts and dodge obstacles while a creature chases you.
- **/yay** — the celebratory "she said yes" page.
- **/garden** — the complete animated flower redesign preserved as a v2 archive.
- **/old** — a **self-contained archive of v1.0** (the original monkey-GIF version),
  kept intact so redesigns never change it.

---

## 🧱 Tech stack

- [Next.js 13](https://nextjs.org/) (App Router) with `output: 'export'` → static HTML
- React 18
- Plain CSS (`app/style.css`) — no CSS framework; animations are hand-rolled
- The two games are **static HTML/CSS/JS apps** served from `public/`
- Deployed via **GitHub Actions** to the `gh-pages` branch

## 🚀 Getting started

Requires a recent **Node.js** (18+).

```sh
npm install       # install dependencies
npm run dev       # dev server at http://localhost:3000
```

### Build the static site

```sh
npm run build     # outputs the static site to ./out
```

Open `out/index.html` (or serve `out/` with any static server) to preview the export.

---

## 🌍 Deployment (GitHub Pages)

Every push to `main` triggers `.github/workflows/gh-pages.yml`, which builds the site
and publishes `out/` to the `gh-pages` branch.

- A `public/.nojekyll` file ships in the export so GitHub Pages serves Next.js's
  `_next/` assets without Jekyll stripping them.
- Because it's a **user page** (`tomas6241.github.io`), assets are served from the
  domain root, so absolute paths (`/fun/...`, `/heart.svg`) work as-is.

## 🎨 Regenerating game sprites (optional)

The Pac-Man-style game's pixel art is generated by helper scripts:

```sh
cd public/fun/web-pacman-main
python3 tools/gen_sprites.py    # regenerate character/heart sprite sheets
python3 tools/gen_scene.py      # regenerate a static scene preview
```
